5. Oscilloscope

The user can inspect signals in running time using the oscilloscope tab.

5.1 Analyzing the plot

The oscilloscope tab contains a plot widget that displays the simulation values at the assigned receiver(s) positions. The values are updated at runtime and the updated front is indicated by a moving green bar, which around the plot. The plot contain up to two lines, one red corresponding to the first receiver (green square) and blue corresponding to the second one (green circle). The two lines are shown overlapped in the same plotting area, to allow for an easier appreciation of phase differences between the receiver signals.

‘Time elapsed’ indicates the value of physical time elapsed in the sumlation.

'Level difference' indicates the sound pressure level difference between the two receivers. By default it shows a (-) value. In order to display an actual level difference, the receiver positions need to be set and the option 'time average (RMS)' in the 'view' tab must be enabled.
'x and y-axis division' indicate the value and corresponding units of the plot division, in the x and y-direction respectively.

5.2 Input receiver probes

In order to set the receiver(s) in the simulation, click with the middle mouse button in the main view.

You can add up to 2 receivers in a given simulation.

When you add a receiver, a green aquare will appear for the first receiver added and a green circle will appear for the second receiver added:

5.2.1 Reset receiver probes

In order to clear the receiver probes data and position, the user needs to press the ‘R’ key.

5.3 Max Lock

Similarly to a real world oscilloscope, the ‘Max. lock’ option locks the y-axis scale to the maximum value reached in a given simulation.

5.4 Save data to disk

The user can save the data in the receiver(s) to disk to allow for post-processing using other software. The output data is written to a 'data.csv' file and is placed inside a folder 'data_output'. If the folder does not exist, it is created in the same location where the executable file is running. The contents of the file 'data.csv' get overwritten every time data is saved, so it is ideal to copy this file to a different folder for post-processing.
