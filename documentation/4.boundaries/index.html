4. Boundaries

The boundaries tab allows the user to:

4.1 Import geometry

This options loads the CAD geometry contained in the file geom.dxf.

As of WAS version 1.x, the .dxf geometry needs to be written to geom.dxf in order to be imported. geom.dxf comes with a default simple drawing, which is used to provide some sense of scale. Ideally, the user should edit the file geom.dxf using a CAD application, making sure the scale of the drawing matches the one used in the example. In order to succeed, the file geom.dxf must contain only lines. This means that any drawing entities (polyline, circles, rectangles) that are not lines will be ignored.

4.2 Add circular/elliptic boundaries

This menu lets the user add a circular boundary to the simulation. The user can choose the radius in the x- and y-directions (in meters), in the eccentricity menu.

Setting a dirrent value for the radius in x and y-directions will naturaly result in a ellipse.

The ‘Scale’ value scales the aforementioned x- and y- radius by an adimensional factor.

4.3 Set boundary absorption coefficient

This option sets the absorption coefficient of boundary conditions. It is based on an algorithm described by Yokota et al[1] and Suzuki et al[2].

In order to set the absorption coefficient of a boundary(s) to a given value, the user must first select them and then click ‘set alpha’. By default, the alpha assigned is zero. This corresponds to a perfetcly rigid boundary, where all the incident sound is reflected and none of it is transmitted.

When the absoption coefficient of a boundary is set to be larger than zero, the boundary will transmit as well as reflect some of the sound incident upon it.
